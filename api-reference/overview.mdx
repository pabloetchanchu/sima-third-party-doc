## Resumen

La referencia interactiva se genera automáticamente desde la especificación OpenAPI, sincronizada diariamente con la colección de Postman de SIMA. Incluye todos los endpoints agrupados por dominio, con schemas de request/response y un **playground en vivo** para probar llamadas directamente desde el browser.

<Info>
  Usá el playground de cada endpoint para probar con tu token. Necesitás autenticarte primero — seguí el [Inicio Rápido](/quickstart) o la guía de [Autenticación](/authentication).
</Info>

## Dominios de endpoints

| Dominio | Versiones | Notas |
|---|---|---|
| Authentication | v2 | Login y refresh token |
| Work Orders | v1 (integración), v3 | Lectura v3; escritura `/integration/api/v1/workOrders` |
| Hierarchy | v2, v3 | Establecimientos, lotes, campañas, cultivos; listado de lotes en v2 |
| Master Data | v2, v3 | Formulados, aplicadores (CRUD v2), máquinas, unidades, tipos de labor |
| Scouting | v2, v3 | Solo lectura; catálogo de adversidades en v2 |
| Progress & Controls | v3 | Avances, controles de siembra/fertilización/stand |
| Silobags | v3 | Solo lectura |
| IoT / Weather | v3 | Dispositivos, eventos, factores climáticos, anomalías |
| Campaign Import | v2 | Importación asíncrona de rendimiento objetivo |
| Tags & Waypoints | v3 | Etiquetas, asociaciones y waypoints de campo |

## Headers requeridos

Cada solicitud debe incluir:

```
Authorization: Bearer {access_token}
X-SIMA-SYSTEM-ID: 8
Accept: application/json
Content-Type: application/json
```
